git status - проверить изменения (лучше выполнять после каждого действия, чтобы удостовериться, что все норм)

1. git pull - выполнить перед началом работы, чтобы проверить не появились ли изменения на удаленной репе
2. Пишешь код
3. git status - должны быть красным цветом выделены файлы, которые изменил
3. git add -A - проиндексировать все изменения
4. git status - красные файлы должны стать зелеными
5. git commit -m "Сообщение о том, что было сделано" - закоммитить изменения
6. git status - вроде такой по итогу ответ должен быть: nothing to commit, working tree clean
7. git push - отправить локальные изменения на удаленный сервер
